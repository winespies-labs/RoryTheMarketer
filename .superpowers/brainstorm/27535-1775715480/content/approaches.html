How should we approach this?

Three ways to implement the nav icons, top bar, and home page polish

A

Targeted Changes Only Recommended

Change exactly the three things we discussed. No collateral refactoring, no new abstractions.

  • Sidebar: add icons, remove redundant "Overview" nav items
  • Layout shell: add sticky top bar component (breadcrumb + page title)
  • Home page: add icons to cards, tighten typography

Pros

  • Fast, low risk
  • Minimal files changed

Cons

  • Doesn't fix other inconsistencies
B

Systematic Pass

Same three changes, plus a light consistency sweep of the other main pages while we're touching layouts.

  • All of A, plus…
  • Standardize spacing and remove inline h1 headers from each page (replaced by top bar)
  • Normalize button styles across Reviews, Swipes, Copywriting pages

Pros

  • More coherent result
  • Fixes things you'll notice later

Cons

  • More files, harder to review
C

Extract Shared Components

Build a small set of reusable UI primitives (PageHeader, NavIcon map) that all pages share.

  • All of A + B, plus…
  • New app/components/PageHeader.tsx used by every page
  • Centralized icon map for nav

Pros

  • Most maintainable long-term

Cons

  • Most work, overkill for a cleanup

I recommend A — targeted, fast, and focused on exactly what we discussed.
